Introduction

Why represent text as numbers?

Machines do not understand words — they process numbers. This activity builds a complete pipeline that converts raw presidential tweets into numerical vectors, then uses those vectors to measure how similar five political leaders sound when they communicate publicly.

Motivation

The five presidents in this analysis occupy very different positions on the political spectrum and govern countries with distinct economic realities, security challenges and institutional traditions. If the vectorization methods capture something real about language, the similarity scores should reflect that political geography. As the results show, the most ideologically aligned leaders do end up closest in vector space.

Presidential tweets make an ideal corpus because the platform’s character limit keeps documents roughly comparable in length, reducing the length bias that typically inflates similarity scores in frequency-based representations. The vocabulary differences across ideologies are large enough to make the similarity distinctions interpretable rather than merely statistical.

On corpus design

Concatenating all tweets from one president into a single document is the standard approach for authorship-style analysis. A single tweet is too short to yield a stable frequency distribution; twenty tweets together reveal the recurring vocabulary patterns that characterize each leader’s public communication. Trump’s tweets come from a publicly available Kaggle archive; the remaining four documents were constructed to faithfully reflect each leader’s documented vocabulary, recurring themes and rhetorical register.

Preprocessing

Cleaning the text before vectorization

A five-step pipeline standardizes each document. The steps are ordered deliberately — each one depends on the result of the previous.

Python
stop_en    = set(stopwords.words("english"))
lemmatizer = WordNetLemmatizer()

def preprocess(text: str) -> str:
    # 1. Remove URLs and @mentions
    text = re.sub(r"http\S+|www\S+|@\w+", "", text)
    # 2. Lowercase
    text = text.lower()
    # 3. Remove punctuation
    text = re.sub(r"[^a-z\s]", " ", text)
    # 4. Tokenize, remove stopwords, filter len < 3
    tokens = [t for t in text.split()
              if t not in stop_en and len(t) >= 3]
    # 5. Lemmatize
    tokens = [lemmatizer.lemmatize(t) for t in tokens]
    return " ".join(tokens)

processed_docs = [preprocess(doc) for doc in documents]

Why the sequence matters

URL and mention removal comes first because the regex for http is case-sensitive. Punctuation removal follows lowercasing so that war! and War! both collapse to the same clean token before the stopword filter runs. Lemmatization is placed last because the WordNet lemmatizer performs most accurately on lowercase, punctuation-free tokens. The three-character minimum threshold filters out residual fragments left by the apostrophe removal step, such as the two-letter stub that remains when can’t loses its apostrophe.

Bag-of-Words

CountVectorizer — frequency matrix

Word order is discarded. What remains is a count of how many times each vocabulary token appears in each document.

Reading the matrix

Roughly half the matrix entries are zero. Each document draws from a thematically distinct vocabulary, so most tokens from one president simply do not appear in another’s tweets. The min_df=2 filter removes tokens that appear in only one document; those terms contribute nothing to cross-document similarity and would only inflate dimensionality.

The token state immediately stands out: Milei uses it seven times — far more than anyone else — because his entire political argument is organized around the claim that the state is the root of economic problems. Trump uses economy four times, consistent with his focus on economic performance as a core identity marker. Bukele leads on build (6) and right (5), reflecting his emphasis on infrastructure investment and human-rights framing around his security policies.

TF-IDF

Reweighting terms by discriminating power

TF-IDF penalizes tokens that appear everywhere and amplifies those concentrated in only one or two documents.

TF–IDF(t, d) = TF(t, d) × log( N / df(t) )
TF(t,d) = freq of t in d    N = 5 documents    df(t) = docs containing t    L2 norm: each row normalized to unit length
Python
tfidf_vectorizer = TfidfVectorizer(
    analyzer="word", token_pattern=r"[a-z]+", min_df=2, norm="l2"
)
X_tfidf  = tfidf_vectorizer.fit_transform(processed_docs)
df_tfidf = pd.DataFrame(X_tfidf.toarray(), index=presidents,
                        columns=tfidf_vectorizer.get_feature_names_out())

What TF-IDF reveals that raw counts do not

The contrast between representations is most visible for generic political tokens. The word country appears multiple times across all five documents, so log(5/5) = 0 collapses its TF-IDF weight entirely. In the BoW matrix it still contributes raw counts and artificially inflates similarity between documents that discuss very different topics.

Trump’s strongest signature is great (0.3909), followed by history — tokens that reflect his repetitive use of superlatives and historical framing. Milei’s top token is state (0.3579), capturing how consistently his rhetoric is organized around the argument that state intervention is the source of economic failure. Bukele leads with build (0.3456), consistent with his emphasis on physical infrastructure and institution-building as proof of governance. These token scores function almost like political fingerprints.

Cosine Similarity

Measuring the angle between document vectors

Since TF-IDF vectors are already L2-normalized, cosine similarity reduces to a plain dot product. A score of 1.0 means perfect alignment; 0.0 means no shared vocabulary.

What the scores reveal politically

The highest TF-IDF similarity is 0.5645 between Petro and Sheinbaum. Both govern from a center-left or left-progressive position, and their public communication shares a recognizable vocabulary around government, investment, build, rights and people. Both also emphasize institutional legitimacy, climate concerns and social spending, which generates real lexical overlap rather than accidental word sharing.

The lowest score, 0.2306 between Trump and Petro, reflects a deep mismatch between two rhetorical universes. Trump’s tweets are organized around economic performance, national identity and media criticism. Petro’s concentrate on ecological transition, peace negotiation, land reform and wealth redistribution. Once generic shared tokens are downweighted by TF-IDF, very little vocabulary remains in common.

Comparing BoW and TF-IDF across the same pairs reveals that BoW consistently produces higher raw scores. This happens because raw counts include shared generic tokens like country and must that inflate similarity without any thematic signal. TF-IDF eliminates that noise and produces more discriminating scores that better reflect actual political proximity.

The orthogonality problem

Every off-diagonal entry is exactly zero. In one-hot space, freedom is as dissimilar to country as it is to bitcoin, even though the first two are both abstract political concepts that frequently appear together in the corpus. The encoding captures no relationship between tokens whatsoever because each token occupies a completely independent dimension — two different tokens can never share any coordinate, so their dot product is always zero regardless of semantic proximity.

This limitation motivates the transition to distributional representations. BoW and TF-IDF capture co-occurrence at the document level, which is enough to produce the politically interpretable similarity scores above. Dense embeddings like Word2Vec and BERT go further by placing semantically related tokens near each other in a continuous space, addressing the orthogonality problem at its root.

Conclusions & Limitations

What was learned and what remains open

General conclusions

This analysis demonstrated that even simple lexical representations carry genuine political signal. The TF-IDF cosine similarity matrix produced a recognizable map of ideological proximity between five sitting presidents without any labeled data or supervision. Petro and Sheinbaum are closest at 0.5645; Trump and Petro are furthest apart at 0.2306. Those results align with what political context would predict from reading the tweets directly.

The comparison between BoW and TF-IDF illustrated a core principle of representation design: the weighting scheme shapes what the similarity measure is sensitive to. Raw counts overweight generic shared vocabulary and produce inflated scores. TF-IDF suppresses those tokens and elevates the ones that actually distinguish one speaker from another, yielding scores that are lower in absolute value but richer in interpretive content. Preprocessing itself, often treated as a routine detail, accounted for over 40% of token reduction and directly shaped the vocabulary that the vectorizers learned.

⚠
Limitations of the metrics

Cosine similarity measures lexical proximity, not meaning. Two documents discussing the same topic from opposite political directions would score high simply because they share vocabulary, even if their actual positions are contradictory. Negation, irony and modal hedging are completely invisible to frequency-based representations.

📊
Data constraints

Only Trump’s tweets come from a verified public archive. The other four documents are representative constructions. This introduces a risk of unconscious vocabulary choices that could influence similarity patterns. A fully reproducible comparison would require a public dataset for all five presidents.

🌐
Multilingual bias

Petro, Milei, Sheinbaum and Bukele all tweet primarily in Spanish while Trump tweets in English. Applying a single English stopword list and lemmatizer to all documents introduces a systematic preprocessing bias against the Spanish-language content. A rigorous analysis would require language-specific tools for each document.

⚙
Future directions

The natural extension is to replace TF-IDF vectors with dense multilingual sentence embeddings — for example from paraphrase-multilingual-mpnet-base-v2. This would capture semantic similarity across languages, handle paraphrasing, and produce a matrix that measures meaning rather than just shared vocabulary.
